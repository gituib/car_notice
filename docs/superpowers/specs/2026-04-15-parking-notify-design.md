# 停车通知系统设计文档

## 1. 项目概述

### 1.1 项目背景
设计一个停车通知系统，通过生成二维码贴在车上，别人扫码后可以通知车主移车。支持多种通知方式，包括钉钉、飞书、企业微信、邮件和短信。

### 1.2 核心功能
- 生成包含通知配置的加密二维码
- 扫码发送通知给车主
- 支持多种通知方式
- 车主可自定义通知内容
- 扫码者可输入简短留言
- 无需数据持久化，配置信息加密嵌入二维码

### 1.3 技术栈
- 后端：Python + Flask
- 前端：HTML5 + CSS3 + JavaScript
- 二维码生成：qrcode 库
- 加密：cryptography 库
- 通知集成：各平台 API

## 2. 系统架构

### 2.1 整体架构

```
┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│   车主用户   │         │   Flask 后端  │         │  通知服务   │
│  (生成二维码) │────────▶│  (API 服务)   │────────▶│  (钉钉/飞书等)│
└─────────────┘         └─────────────┘         └─────────────┘
                              ▲
                              │
┌─────────────┐               │
│  扫码用户    │───────────────┘
│  (发送通知)  │
└─────────────┘
```

### 2.2 模块划分

#### 后端模块
1. **API 模块**：处理前端请求，路由管理
2. **加密模块**：负责配置信息的加密和解密
3. **二维码模块**：生成和解析二维码
4. **通知模块**：实现各种通知方式的发送逻辑
5. **配置模块**：管理系统配置和环境变量

#### 前端模块
1. **生成二维码页面**：车主配置通知信息并生成二维码
2. **通知页面**：扫码后发送通知的界面
3. **公共组件**：复用的 UI 组件和工具函数

## 4. 核心模块设计

### 4.1 加密模块

**职责**：
- 对车主配置信息进行加密
- 对二维码中的加密信息进行解密
- 确保数据安全

**实现**：
- 使用 AES-256-GCM 对称加密
- 密钥存储在环境变量中
- 加密数据包含：通知配置、通知内容、时间戳

### 4.2 二维码模块

**职责**：
- 生成包含加密数据的二维码
- 支持彩色二维码样式
- 提供二维码下载功能

**实现**：
- 使用 `qrcode` 库生成二维码
- 支持自定义颜色
- 生成 base64 编码的二维码图片

### 4.3 通知模块

**职责**：
- 实现各种通知方式的发送逻辑
- 提供统一的通知发送接口
- 处理通知失败的情况

**实现**：
- 采用策略模式，定义通知基类
- 各种通知方式继承基类实现具体逻辑
- 支持批量发送通知

### 4.4 API 模块

**职责**：
- 处理前端请求
- 提供 RESTful API 接口
- 验证请求参数

**实现**：
- 使用 Flask 路由
- 提供以下接口：
  - `/api/generate` - 生成二维码
  - `/api/notify` - 发送通知
  - `/api/health` - 健康检查

### 4.5 前端模块

**职责**：
- 提供用户友好的界面
- 处理用户输入
- 与后端 API 交互

**实现**：
- 响应式设计，适配不同设备
- 表单验证
- 异步 API 调用

## 5. 数据流程

### 5.1 生成二维码流程

1. 车主访问生成二维码页面
2. 填写通知配置（钉钉、飞书等）
3. 自定义通知内容
4. 点击生成二维码
5. 前端发送配置到后端 API
6. 后端加密配置信息
7. 后端生成二维码
8. 前端显示二维码并提供下载

### 5.2 扫码通知流程

1. 扫码用户扫描二维码
2. 浏览器打开通知页面，解析二维码数据
3. 扫码用户输入简短留言（可选）
4. 点击发送通知
5. 前端发送请求到后端 API
6. 后端解密配置信息
7. 后端调用相应的通知服务
8. 后端返回发送结果
9. 前端显示发送状态

## 6. 安全设计

### 6.1 数据加密
- 使用 AES-256-GCM 加密算法
- 密钥存储在环境变量中，不硬编码
- 加密数据包含时间戳，防止重放攻击

### 6.2 API 安全
- 实现请求频率限制
- 验证请求参数格式
- 处理异常情况，不暴露敏感信息

### 6.3 隐私保护
- 不存储车主配置信息
- 通知内容加密传输
- 扫码者只能发送通知，无法获取车主详细信息

## 8. 技术选型理由

### 8.1 后端选择 Python + Flask
- **轻量级**：Flask 是轻量级框架，适合快速开发
- **灵活性**：易于扩展和定制
- **生态丰富**：有大量第三方库支持二维码生成、加密等功能
- **易于部署**：可以在各种平台上运行

### 8.2 前端选择纯 HTML/CSS/JavaScript
- **轻量**：加载速度快，适合扫码场景
- **兼容性**：支持各种浏览器
- **无需构建**：直接部署，简化开发流程
- **响应式**：适配不同设备屏幕

### 8.3 加密选择 AES-256-GCM
- **安全性高**：256 位密钥，GCM 模式提供认证
- **性能好**：加密解密速度快
- **标准算法**：广泛使用，经过安全验证

## 10. 结论

本设计采用模块化架构，使用 Python + Flask 后端和纯前端技术栈，实现了一个轻量级、安全的停车通知系统。系统不需要数据持久化，通过加密将配置信息嵌入二维码中，既保证了安全性，又简化了部署和维护。

模块化的设计使得系统易于扩展和维护，未来可以根据需求添加新的功能和通知方式。同时，系统支持多种部署方式，适应不同的使用场景。
